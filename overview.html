Vital Patient Data Insights with SQL

This project is a SQL-based hospital management system designed to manage and analyze patient, doctor, and appointment data efficiently. It demonstrates essential SQL skills like table creation, foreign key relationships, and advanced queries for data analysis.

Project Overview:

  • Patients Table: Stores patient details like first name, last name, and gender.
  • Doctors Table: Contains doctor information, including their specialties.
  • Appointments Table: Tracks appointments, linking patients and doctors with appointment dates and statuses.

Key SQL Concepts Demonstrated:

  • Creating and managing relational database tables.
  • Using foreign keys to establish relationships between tables.
  • Executing complex queries to retrieve, group, and analyze data.

Sample Queries:

-- Get all scheduled appointments
SELECT * FROM Appointments WHERE status = 'Scheduled';

-- Count patients by gender
SELECT gender, COUNT(*) AS patient_count FROM Patients GROUP BY gender;

-- Join Patients and Appointments to see appointment details
SELECT P.first_name, P.last_name, A.appointment_date, A.status
FROM Patients P
LEFT JOIN Appointments A ON P.patient_id = A.patient_id;

This project showcases the power of SQL in managing real-world healthcare data and provides a strong foundation for database management and data insights.
